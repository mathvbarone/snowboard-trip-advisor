## Project context (3 sentences)

Snowboard Trip Advisor is a **data-transparency comparison tool** for European ski resorts. The product surfaces durable resort facts and live market signals **side-by-side with visible source provenance**, and the user does the ranking themselves ‚Äî there is no scoring layer. Phase 1 is EUR-displayed (with FX provenance for non-Eurozone upstreams), EU-only, two seed resorts, filesystem persistence, no auth, loopback admin only.

## Deviations from the spec to remember (do not silently revert)

These were captured during Epic 1 + Epic 2 execution. Each has a specific reason:

1. **`AdapterSourceKey` ‚äÇ `SourceKey` split** (`packages/schema/src/primitives.ts`). 5 adapters in `AdapterSourceKey`; `SourceKey` adds `'manual'` for editorial fixture entries. The `Registry` type uses `AdapterSourceKey`; `FieldSource.source` uses `SourceKey`.
2. **Stubs publish their actual adapter identity in `FieldSource.source`**, not `'manual'`. Stub-ness is signalled by empty `values:{}`, all-zero `STUB_HASH`, `source_url: 'https://example.invalid/stub'`, and a stub-marked `attribution_block`.
3. **`AdapterValueMap = Partial<Record<MetricPath, unknown>>`** instead of the spec's tighter shape. Per-field tightening lands per real adapter in Epic 5.
4. **Root `vitest.config.ts` owns coverage thresholds.** Per-workspace `vite.config.ts` files own `coverage.exclude` lists; their `thresholds` blocks were dropped (vitest workspace mode doesn't aggregate per-project thresholds).
5. **CLI-split pattern.** Side-effect entry scripts (e.g. `scripts/generate-tokens.cli.ts`) are split from pure logic (`scripts/generate-tokens.ts`). Pure logic is unit-tested at 100%; CLI shim is excluded with a written rationale and throws if imported instead of invoked. Apply this pattern to any future generator/runner.
6. **Per-package barrel `index.test.ts`** is a real smoke test asserting every public re-export is defined. Don't delete ‚Äî v8 instruments `index.ts` regardless of imports.
7. **`scripts/pre-commit` install path** uses `git rev-parse --git-path hooks` so `npm run setup` works from worktrees.
8. **ESLint config has a fixture-filename override block** (`**/__eslint_fixture__.{ts,tsx}`) for the fixture-driven lint test.
9. **Branch protection on `main`: `required_pull_request_reviews` REMOVED** (single-maintainer self-approval deadlock). Restoration recipe (when a second maintainer joins): one `PUT /repos/{owner}/{repo}/branches/main/protection` call. Original protection saved at `/tmp/main-protection-pre-relax.json`.
10. **`scripts/apply-branch-protection.sh` is missing** (lost in pre-Epic-1 cleanup); restoration is an Epic 6 polish task.
11. **`research/` directory does not exist yet.** Spec assumes it (¬ß7.9). Epic 5 PR 5.1 reintroduces `research/cli.ts`.
12. **Apps are visual placeholders.** Both `apps/{public,admin}/src/App.tsx` render `<main className="app-shell" />`. Epic 3 / Epic 4 fill them in.
13. **`vitest.config.ts` `InlineConfig` type** is imported from `'vitest/node'` (the maintained path).
14. **Per-workspace `tsconfig.json` does NOT set `rootDir`.** Setting `rootDir: "src"` while `include: ["src", "vite.config.ts"]` triggers TS6059 on per-workspace `npm run typecheck --workspace=...`.
15. **`.github/workflows/image.yml` is paused.** PR #8 removed it from the post-merge dispatch; the Dockerfile is pre-pivot. Re-enable with `on: release: { types: [published] }` once Dockerfile + nginx.conf + image.yml are reworked for the workspace layout (Epic 6).

## Process discipline (non-negotiable)

- **TDD**: failing test ‚Üí minimal implementation ‚Üí verify pass.
- **DCO**: `git commit -s` on every commit. Never `--no-verify`.
- **100% coverage hard gate**. Coverage exclusions live in workspace `vite.config.ts` with a written rationale. No `/* istanbul ignore */`. The project has used `/* v8 ignore */` markers in two genuinely-unreachable defensive branches in `publishDataset.ts` (macOS-APFS dir-fsync EBADF tolerance + lock-timeout throw); each is paired with rationale in both the source and `packages/schema/vite.config.ts`. Treat that as the precedent if you need similar suppression ‚Äî but reach for *test the branch* first, *file-level exclude* second, *inline marker* last.
- **No `console`** outside `scripts/**` (and once `research/` exists, `research/cli.ts`, `research/migrate/**`).
- **Pre-commit hook runs `npm run qa`**.
- **Subagent Review Discipline**: any PR touching CODEOWNERS-listed paths (CLAUDE.md, spec, ADRs, `.github/`, hooks, eslint config, `packages/schema/**`, `packages/schema/api/**`) gets an independent subagent code review before merge. Required-CODEOWNER-review on `main` is **off** in Phase 1 (single-maintainer); this is a maintainer practice rather than a hard merge block.
- **Worktrees**: use `superpowers:using-git-worktrees`. `.worktrees/` and `.claude/worktrees/` are gitignored.
- **Plan-first**: use `superpowers:writing-plans` against the spec for any non-trivial epic; review-loop until approved. Run the `ai-clean-code-adherence` audit before declaring the plan ready.
- **Subagent-driven execution**: `superpowers:subagent-driven-development` ‚Äî fresh implementer per task, then spec-compliance review, then code-quality review, then mark done.
